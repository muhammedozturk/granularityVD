# granularityVD

	This page introduces a novel granularity-based vulnerability detection (VD) method that addresses critical limitations in existing deep learning and Large Language Model (LLM) approaches. Unlike current techniques that often focus on a single programming language, rely on computationally expensive graph-based processing, or struggle with repo-level analysis, our proposed method utilizes a lightweight, language-agnostic pipeline. It harvests code from GitHub at function, class, and repository levels across C, C++, C\#, and Java.
	
	A key innovation is a risk-sensitive labeling mechanism that correlates GloVe-generated code embeddings with a curated vulnerability keyword corpus to generate training labels. We use keywords by conducting a rigorous manual analysis on Common Weakness Enumeration (CWE). A deep neural network is then trained and fine-tuned via grid search hyperparameter optimization. Experimental results demonstrate that the method achieves strong performance at the function and class levels, with F1-scores exceeding 0.96 for C\# and C. The analysis also reveals significant differences in computational efficiency and hyperparameter sensitivity across programming languages and granularities. This work demonstrates that carefully configured deep learning remains a robust, efficient alternative to LLMs for multi-granularity VD, offering valuable insights into the practical factors influencing detection system performance.
